3.3. Delegación de eventos

Delegación de eventos.

La delegación de eventos consiste en registrar un evento a un elemento para que sean sus elementos hijos los que lo disparen. Esto es necesario cuando los elementos a los que se quiere registrar el evento NO EXISTEN inicialmente en la web.

Ejemplo: Sea una página provista de una lista <ul> cuyos elementos <li> muestran su texto en la consola de desarrollador al hacer click sobre ellos, y un botón “ AÑADIR” que añade nuevos elementos <li> cada vez que se pulsa: 


<!DOCTYPE html>
<html>
<head>
    <title></title>
    <meta charset="utf-8" />
    <style>
        li { cursor: pointer; }
    </style>
    <script src="scripts/jquery-3.2.1.min.js"></script>
    <script>
        $().ready(function () {
            // Registro de evento de pulsación para cada elemento de la lista
            $("ul li").on("click", function () {
                console.log("HAS PULSADO: " + $(this).text()); 
            })
            $('button').click(function () {
                // Añadido de elemento <li> a la lista
                var cantidad = $("ul li").length + 1; 
                $('ul').append($('<li>', {text: 'opcion ' + cantidad})); 
            });
        });
    </script>
</head>
<body>
    <ul id="lista">
        <li>opcion 1</li>
        <li>opcion 2</li>
        <li>opcion 2</li>
    </ul>
    <button>REGISTRAR</button>
</body>
</html>


El registro del evento click a los elementos <li> se ejecuta al cargarse la página.

Sólo los elementos <li> presentes inicialmente en el HTML de la página responden al evento click, pero no los nuevos elementos añadidos dinámicamente al pulsar el botón “ACCION”. Esto es debido a que  los eventos asignados a los elementos de una selección, sólo se asignan a los elementos existentes en ese instante. Los elementos añadidos posteriormente no reciben el registro del evento.

Para vincular funciones manejadoras a eventos de elementos hijos aún no existentes puede emplearse la delegación de eventos. Esto consiste en registrar un evento sobre un elemento para que lo lancen sus elementos hijos tanto actuales como los que puedan ser añadidos dinámicamente por código. 

Esto requiere el uso de la función $().on() vista anteriormente, pero con un argumento más:  

$(<padre>).on(<evento>,<hijos_delegacion>,<manejadora> )

Por lo tanto, para resolver el ejemplo anterior debemos emplear la función $().on() para delegar el evento ‘click’ desde elemento padre <ul> en sus elementos hijos <li>: 

<!DOCTYPE html>
<html>
<head>
    <title></title>
    <meta charset="utf-8" />
    <style>
        li { cursor: pointer; }
    </style>
    <script src="scripts/jquery-3.2.1.min.js"></script>
    <script>
        $().ready(function () {
            // Delegacion de evento de pulsación para los elementos <li> de la lista <ul>
            $("ul").on("click", "li", function () {
                console.log("HAS PULSADO: " + $(this).text()); 
            })
            $('button').click(function () {
                // Añadido de elemento <li> a la lista
                var cantidad = $("ul li").length + 1; 
                $('ul').append($('<li>', {text: 'opcion ' + cantidad})); 
            });
        });
    </script>
</head>
<body>
    <ul id="lista">
        <li>opcion 1</li>
        <li>opcion 2</li>
        <li>opcion 3</li>
    </ul>
    <button>AÑADIR</button>
</body>
</html>

 
Si ahora cargamos la página y añadimos tres elementos <li> a la lista, veremos que al pulsarlos TODOS muestran sus valores en la consola de desarrollador, tantos los elementos <li> iniciales como los añadidos posteriormente mediante el empleo de la delegación:



Propagación de Eventos.

La propagación de eventos es una característica por la que los eventos de los elementos hijos se propagan a los elementos padre. Este fenómeno se aprecia cuando varios elementos contenidos unos dentro de otros capturan los mismos eventos.

Ejemplo: Sea una página que consta de dos capas anidadas en las que se captura el evento de ‘click’ de modo que cada una muestra un mensaje en la consola de desarrollador al hacerse clic sobre ella: 


<!DOCTYPE html>
<html>
<head>
    <title></title>
    <meta charset="utf-8" />
    <style>
        div { border: 1px solid #000; }
    </style>
        $().ready(function () {
            // Registro de evento 'click' en capa padre
            $('#padre').click(function(event) {
                console.log("PADRE PULSADO"); 
            });
            // Registro de evneto 'click' en capa hija
            $('#hija').click(function(event) {
                console.log("HIJA PULSADO"); 
            });
        });
    </script>
</head>
<body>
    <div id='padre'>
        <br/>
        CAPA PADRE
        <br/>
        <div id='hija'>
            <br/>
            CAPA HIJA
            <br/>
        </div>
    </div>
</body>
</html>

Cancelación de la propagación


La propagación de los eventos es algo que sucede de manera predeterminada. Sin embargo, puede que no deseemos este tipo de comportamientos. En tal caso, podemos interrumpir la propagación de un evento al elemento padre, se si en la función manejadora  se invoca el método stopPropagation() del objeto ‘event’ recibido como argumento. 

Ejemplo: El siguiente código modifica la función vinculada al evento click de la capa hija para evitar que el evento se propague a la capa padre:  

<!DOCTYPE html>
<html>
<head>
    <title></title>
    <meta charset="utf-8" />
    <style>
        div {
            border: 1px solid #000;
        }
    </style>
    <script src="scripts/jquery-3.2.1.min.js"></script>
    <script>
        $().ready(function () {
            // Registro de evento 'click' en capa padre
            $('#padre').click(function(event) {
                console.log("PADRE PULSADO"); 
            });
            // Registro de evneto 'click' en capa hija
            $('#hija').click(function(event) {
                console.log("HIJA PULSADO"); 
                // Cancelacion de propagación del evento a la capa padre. 
                event.stopPropagation();
            });
        });
    </script>
</head>
<body>
    <div id='padre'>
        <br/>
        CAPA PADRE
        <br/>
        <div id='hija'>
            <br/>
            CAPA HIJA
            <br/>
        </div>
    </div>
</body>
</html>


Si ahora pulsamos sobre el área de la capa hija con el párrafo "CAPA HIJA" observamos que sólo se ejecuta la función manejadora vinculada al evento clic de la capa hija, pero no el de la capa padre: 



Cancelación de acción predeterminada.


Algunos eventos tienen funcionalidades asociadas que se realiza al margen de lo que hagan las funciones vinculadas. Por ejemplo: 

  • El evento de submit de un formulario se lanza cuando el usuario hace clic sobre el botón de tipo "submit", y después el formulario envíe los datos a la URL indicada en el atributo action de la etiqueta <form>. 
  • El evento keypress de una caja de texto se lanza cuando el usuario pulsa una tecla estando el foco situado sobre una caja de texto, y después la caja de texto muestra la tecla pulsada por el usuario.

En ciertas ocasiones resulta no obstante necesario cancelar la acción predeterminada, como por ejemplo; enviar un formulario si no se cumplen ciertas condiciones sobre los valores introducidos a pesar de que el usuario ha pulsado el botón de envío.  Para cancelar la acción predeterminada de un evento debe llamarse al método preventDefault() del parámetro event recibido en la función manejadora.

Ejemplo: Sea una página web provista de un formulario con una caja de texto y un botón de envío. Se desea evitar que el formulario se envíe si el usuario deja vacía la caja de texto, en cuyo caso debe mostrarse un mensaje de error:


El envío del formulario se controla registrando el evento “submit” del formulario. En la función manejadora se comprueba el contenido de la caja de texto y se invoca la función preventDefault() del parámetro ‘event’ si  está vacía para evitar la acción predeterminada de enviar el formulario: 

<!DOCTYPE html>
<html>
<head>
    <title></title>
    <meta charset="utf-8" />
    <script src="scripts/jquery-3.2.1.min.js"></script>
    <script>
        $().ready(function () {
            // Registro del evento de envío del formulario
            $('#formulario').submit(function(event) {
                // Comprobacion de contenido de la caja de texto
                if ( $('input:text').val() == "" )
                    alert("No hay valor"); 
                    // Cancelacion del comportamiento predeterminado
                    event.preventDefault();
            });
        });
    </script>
</head>
<body>
    <form id='formulario'>
        <input type='text'>
        <input type='submit'>
    </form>
</body>
</html>
